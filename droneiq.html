Context

Odyssey2.0 is a Management Information System in the North American sector. My contributions encompassed three core aspects: first, crafting a comprehensive User Interface kit to strengthen the digital brand; second, developing an integrated design system by merging code components; and finally, serving as a User Interface designer. This ensured efficient development processes, higher scalability rates, and a cohesive, user-centric interface.

Problem.

This startup was experiencing rapid growth and was hiring many new members without having unified identity documents. This situation resulted in several problems: numerous inconsistencies across the application, a steep learning curve for newcomers, reduced scalability, and a delayed rollout of new components from design to development. All of these issues directly impacted the overall pace of the process.

User Research

Low fidelity mock-ups

UI Kit

Solution.

Establishing a design system would address the inconsistency issues, accelerate scalability rates, and serve as a foundation for building components within the application. We held a meeting with the Design System Principal at Oracle to gain a better understanding of how to tackle this significant challenge. Based on that meeting, we outlined a roadmap for creating our own design system.

Process.

So, alongside two front-end developers and two UX designers, I led the initiative to create the design system. This involved cataloging every single component used in the application, which included documenting rules and guidelines, providing documentation about the components, linking to the UI Kit in Figma, and organizing the Scss class components.

Integration of APIs

Presenting to stakeholders

Public website

Results (metrics).

When we presented the prototype along with all the documentation, we not only met the expected timeline but also, through extensive user research, managed to reduce expenses by almost a third of the original cost. The documentation would also prove invaluable for scalability and serve as a foundation for designers and developers as the project progressed.
